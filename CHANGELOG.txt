==================

Database:
- Clients should have their key be a unique ID (integer). Names should still be unique, but not the key.
- Session duration should be stored in minutes.
- Sessions should have a unique ID as their key instead of name+date+time.
- Sessions should have a new optional field called "rate" representing an adjusted rate.

Routes:
- Routes should now use client and session ID as endpoints: /api/clients/{client_id}, etc.
- /api/clients/:name should also work the same as /api/clients/:client_id (names are unique).

Frontend:
- Client editing should allow changing the client name.
- New session form and edit session form should have a spot for rate.
- Durations and sum of durations (in clients/summaries/payments) should be in the format XhXm.
- Revenue and payment calculations must take into account the possible adjusted rate for sessions.
- When clicking a client in the payment view, the default value for the payment should be the amount they owe.

Other:
- Remove the requirement for a single payment per client per day (no UNIQUE constraint on client+date in payments).
- Write a migration script to update the Railway DB to the new schema without losing any data.

migrate_railway.js (new file)
- Standalone Node.js script to migrate an existing Railway DB in-place
- Checks if already migrated (exits early if clients.id exists)
- Snapshots row counts before migration
- In a single transaction:
    1. Renames clients, sessions, payments to *_old
    2. Creates new tables with updated schema
    3. Inserts clients (SQLite assigns new sequential IDs)
    4. Inserts sessions: joins on client name to get client_id,
       converts duration with CAST(ROUND(duration * 60) AS INTEGER),
       backfills rate from client's rate
    5. Inserts payments: joins on client name to get client_id
    6. Drops the *_old tables
- Verifies row counts match before/after; exits with error if mismatch
- Usage: DB_PATH=/path/to/scheduler.db node migrate_railway.js

client/src/components/SessionModal.js — DurationInput component
- Replaced two-field hours/minutes inputs with a single H:MM segment input
- Clicking or tabbing into each segment highlights it; typing updates that segment
- Hours: single digit only (0–9); auto-advances to minutes on keypress
- Minutes: two digits (00–59); accepts first digit then waits for second
- ArrowUp/Down increment/decrement the active segment (minutes in steps of 5)
- ArrowRight/Left navigate between segments
- Tab/colon advance hours→minutes; Shift+Tab retreats minutes→hours
- Default duration is 1:00 for new sessions
- Minimum duration of 30 minutes enforced on submit
